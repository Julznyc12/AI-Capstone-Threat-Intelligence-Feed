# Component 1 – Threat Intelligence Feed Collection

## Overview

Component 1 is responsible for collecting cybersecurity threat intelligence articles from multiple RSS feed sources and storing the results in Airtable for downstream processing.

This component serves as the data ingestion layer of the AI Threat Intelligence Feed Dashboard project. It automates the retrieval of newly published threat intelligence content and creates a centralized dataset that will be enriched and analyzed by later workflow components.

---

## Objective

The purpose of this component is to:

- Collect cybersecurity threat intelligence articles from RSS feeds
- Normalize incoming data into a consistent structure
- Automatically store records in Airtable
- Eliminate manual data collection
- Provide fresh threat intelligence data for AI processing

---

## Technologies Used

- n8n
- RSS Feed Trigger
- Airtable
- JavaScript
- HTTP Requests

---

## Workflow Architecture

```text
Schedule Trigger
        ↓
RSS Feed Sources
        ↓
Normalize Article Data
        ↓
Airtable Record Creation
```

### Step 1: Schedule Trigger

The workflow begins with a scheduled trigger that automatically executes the pipeline at predefined intervals.

Purpose:

- Automates threat intelligence collection
- Eliminates manual execution
- Ensures data remains current

---

### Step 2: RSS Feed Collection

Multiple cybersecurity RSS feeds are queried to retrieve the latest threat intelligence articles.

Examples of collected information include:

- Article title
- Publication date
- Source
- Description
- Link to original article

Purpose:

- Gather current cybersecurity news
- Collect threat intelligence information
- Build a centralized feed repository

---

### Step 3: Data Normalization

Incoming RSS data is transformed into a consistent structure before storage.

Normalized fields include:

| Field | Description |
|---------|---------|
| Title | Article title |
| Source | RSS source |
| Link | Original article URL |
| Published Date | Publication timestamp |
| Summary | Article description |
| Collection Timestamp | Workflow execution time |

Purpose:

- Standardize records from different sources
- Improve downstream AI processing
- Maintain consistent database formatting

---

### Step 4: Airtable Storage

Processed records are inserted into Airtable.

Airtable acts as the project's central data repository and stores all collected threat intelligence articles.

Purpose:

- Centralize collected data
- Enable later enrichment workflows
- Support dashboard visualizations

---

## Inputs

### RSS Feed Data

Input data originates from cybersecurity RSS feeds and contains:

- Article titles
- Publication dates
- Descriptions
- URLs
- Source information

---

## Outputs

### Airtable Records

Each collected article is stored as a structured Airtable record containing:

- Threat article title
- Publication date
- Source
- Summary
- URL
- Processing metadata

These records become the input for Component 2.

---

## Business Value

This component provides:

- Automated threat intelligence collection
- Consistent cybersecurity data ingestion
- Reduced analyst workload
- Centralized data management
- Scalable collection architecture

Without this component, analysts would need to manually search multiple sources and enter information into the system.

---

## Component Deliverables

- Automated RSS collection workflow
- Structured Airtable database population
- Threat intelligence article repository

## Next Component

Component 2 consumes the Airtable records generated by Component 1 and applies AI-powered enrichment, classification, and analysis to each collected threat intelligence article.
